# 00 面试题说明

本目录用于保存真实面试问题的来源、核验状态、题目归类和高频统计。

## 核验状态

- `VERIFIED`：正文可访问，能够确认是候选人实际面试记录，且正文出现该问题。
- `PARTIAL`：只能确认部分信息，例如正文需要登录、只能看到摘要、轮次缺失或来源为二手报道。
- `UNVERIFIED`：无法确认是真实面经。

章节文档中的“真实面试问题”优先收录 `VERIFIED` 和 `PARTIAL`。`UNVERIFIED` 只能作为候选问题指针出现，完整记录必须放到 `未核实候选问题.md`。

## 每条问题记录格式

```text
### 问题：……

* 公司：
* 岗位：
* 轮次：
* 面试时间：
* 技术模块：
* 核验状态：
* 原帖标题：
* 原帖作者：
* 来源 URL：
* 发布时间：
* 访问日期：
* 原帖表达：短摘录或忠实转述
* 本题考察点：
* 建议回答框架：
* 可能追问：
* 对应学习章节：
* 尚未确认的信息：
```

不要大段复制原文。

## 出现频次分层

### 已出现问题

至少 1 个 `VERIFIED` 或 `PARTIAL` 真实面经样本出现过该问题或等价问法。

### 多次出现问题

至少 2 个相互独立的真实面经样本出现过该问题或等价问法。

### 高频问题

只有满足以下条件，才能使用“高频面试问题”：

- 至少 3 个相互独立的真实岗位面经样本；
- 三个样本不是同一篇汇总文章拆分出来的；
- 记录样本来源；
- 记录统计时间范围；
- 说明这只是公开面经样本，不代表完整行业统计。

若只有 1 到 2 个来源，只写“已出现问题”或“多次出现问题”，不得称为“高频”。

## 网站范围

面试题来源包括但不限于以下平台：

- 牛客网；
- 知乎；
- 小红书；
- 脉脉；
- GitHub Issue / Discussion；
- 掘金、博客园、CSDN、个人技术博客；
- 公众号文章、公司技术社区、论坛帖子；
- 公开搜索结果中能继续追溯到原帖的页面。

平台本身不决定可信度。是否收录取决于能否确认候选人真实面试经历、公司或团队、岗位方向、轮次、问题原文或忠实转述、页面来源和访问日期。

对需要登录、只能看到摘要、截图或二手转述的内容，要按证据强弱标记为 `PARTIAL` 或 `UNVERIFIED`，不得为了凑题量写成 `VERIFIED`。
